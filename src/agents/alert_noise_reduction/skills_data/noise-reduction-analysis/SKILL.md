# 告警降噪分析

## 流程
1. 调用 `search_similar_alerts(alert_content="[告警内容]", top_k=5)`
2. 根据历史数据判定噪音等级
3. 按模板输出报告
<!-- 4. 查询天气 -->

## 噪音判定

| 等级 | ACK率 | 历史根因 | 业务影响 |
|------|-------|----------|----------|
| 噪音 | >30% | 已知/计划修复 | 无 |
| 疑似噪音 | 20-30% | 间歇性 | 低 |
| 非噪音 | <20% | 未知 | 高 |

**强制非噪音**：安全告警、数据丢失、核心业务中断、无历史记录

## 输出模板

```markdown
## 分析当前报警

- **事件摘要**: [噪音/疑似噪音/非噪音] | [未处理/已ACK] | 通知[N]次
- **Agent分析结果**:
  - **产生原因**: [基于历史的根因分析]
  - **推荐SOP**:
    - [步骤1]
    - [步骤2]
  - **判断逻辑**: [ACK率+历史特征+tags一致性]
- **历史报警信息**:
  - **发送统计**: 过去[N]天发了[M]次
  - **处理统计**: ACK比例[X]%
  - **最相似记录**: [时间] [根因]
  - **Tags一致性**: [一致/不一致]
  - **历史知识**: [经验总结]
```

## 无历史数据

```markdown
## 分析当前报警

- **事件摘要**: 非噪音 | 未处理 | 通知[N]次
- **Agent分析结果**:
  - **产生原因**: 无历史数据，无法确定
  - **推荐SOP**:
    - 按标准流程处理
    - 记录处理过程
  - **判断逻辑**: 无历史参考，保守处理
- **历史报警信息**:
  - **发送统计**: 无
  - **处理统计**: 无
  - **最相似记录**: 未找到
  - **Tags一致性**: 无法对比
  - **历史知识**: 新告警，建议记录
```
